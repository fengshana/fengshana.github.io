中国被世界黑的最惨的一天

上午一开始，公众提交申请空前踊跃。
上午9时至10时，官方票务网站的浏览量达到了800万次，票务呼叫中心热线从9时至10时的呼入量超过了380万人次。

京东促销爆服务器 刘强东：再搞一次

  • 人员过多
  • 访问量比较大
  • 流量过大

问题现象

表层现象

  • 海量用户
  • 高并发（所有人一块进来访问网站）

核心问题

罪魁祸首—–关系型数据库

数据访问中影响因素：

  • 性能瓶颈：磁盘IO性能低下
    • 数据库是保存在磁盘上的，而磁盘的IO性能也就是读写性能相对来说比较低下；
    • CPU的数据交换是和高速缓存Cache来做；
    • Cache的数据交换和内存做；
    • 内存的数据交换和硬盘做；
    • 提供基础数据的这些信息在硬盘上存储；而且是一层一层的往上进行传递；导致性能低下；
  • 扩展瓶颈：数据关系复杂，扩展性差，不便于大规模集群
    • 数据库中数据的存储格式；
    • 数据库不仅仅存储数据；更多的还需要进行描述数据关系（表与表之间的多关联）；（呈现网状关联，前后交叉，错综复杂，牵一发而动全身）

解决思路

NoSQL
  • 降低磁盘IO次数，越低越好（针对IO性能差） ——- 内存存储
  • 去除数据间关系，越简单越好 ——– 不存储关系，仅存储数据

NoSQL: 即 Not-Only SQL （泛指非关系型的数据库），作为关系型数据库的补充；

Not 不；
Only 仅仅；只是；
Not Only 不仅仅，不只是；
最终指向的即为做事情不能够仅仅只用SQL；与数据存储访问的类似的泛指称作是NoSQL；

作用： 应对基于海量用户和海量数据前提下的数据处理问题（高并发）；

大量用户对大量数据访问出现的一个集中问题；

特征：
（特征都是相对的；没有绝对的）

  • 可扩容，可伸缩；
  • 大数据量下高性能
  • 灵活的数据类型（能兼容数据库中的数据）
  • 高可用
